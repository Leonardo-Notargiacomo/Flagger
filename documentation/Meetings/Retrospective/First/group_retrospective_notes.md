---

### 1. Communication & Collaboration
- The team agreed to use **Discord** as the primary communication channel for updates, questions, and progress tracking.
- Members should **post weekly updates** summarizing what they’re working on and what they’ve completed.
- Encourage **constructive engagement**—react to others’ updates, but avoid public criticism. If issues arise, discuss them privately.
- Proposed creating separate Discord threads or channels for:
  - Current work updates
  - Review requests
  - Challenges & help requests

**Key Action Points:**
- Everyone posts at least one update per week.
- Refrain from judging updates publicly; communicate issues privately.
- Create a GitHub review channel or similar Discord space for pull request notifications.

---

### 2. Planning & Documentation
- Recognized need for **better upfront planning** and **clearer documentation**.
- Suggestion to create **basic diagrams (class, domain, or relational schemas)** to align on architecture decisions.
- The group agreed that too much planning can hinder agility, but **fundamental structure is necessary** for coordination.
- There’s confusion around current progress—members are unsure who is doing what at times.
- Decision: All major discussions (e.g., diagrams, color schemes, backend relations) must have a **documented conclusion on GitHub**.

**Key Action Points:**
- Introduce minimal but consistent documentation standards.
- Create an `artifacts/` folder on GitHub with subfolders for `meetings/` and `documents/`.
- Link documentation to relevant user stories or issues on the Kanban board.

---

### 3. Task Division & Agile Practices
- Mixed opinions on task planning: some prefer implementing quickly, others favor structured analysis and design first.
- Consensus reached: follow a **lightweight analysis–design–implementation** process:
  1. Define user story & use case.
  2. Create necessary diagrams (if applicable).
  3. Implement feature.
  4. Document outcomes briefly.
- Emphasis on **small, iterative deliverables** rather than long ad-hoc work.

**Key Action Points:**
- Define how each new feature will be approached (analysis/design first, then implementation).
- Document and link artifacts for every feature.
- Ensure clarity on individual responsibilities via Kanban or Discord.

---

### 4. Experimentation & Learning
- Agreed that experimentation (e.g., testing new features like push notifications or maps) is valuable but must be **structured**.
- Suggest following a scientific approach: plan → experiment → observe → conclude → document.
- Experiments should include brief written reflections or documentation for others to reference.

**Key Action Points:**
- Use a template for recording experiments (goal, steps, results, conclusion).
- Store in `/artifacts/experiments/` on GitHub.

---

### 5. Efficiency & Focus
- Discussion on time management: avoid spending excessive time on unrelated topics or entertainment during work sessions.
- Maintain a balance between relaxed atmosphere and focus.

**Key Action Points:**
- Minimize off-topic conversations during group sessions.
- Keep meetings structured and timeboxed.

---

### 6. Continuous Delivery & Git Practices
- General satisfaction with Git workflow but identified room for improvement in **merge frequency and clarity**.
- Members should:
  - Work on feature branches.
  - Commit and push frequently.
  - Request reviews through GitHub or Discord.
  - Maintain clean commit messages.

**Key Action Points:**
- Standardize commit and branching practices.
- Encourage peer reviews and documentation of merges.

---

### 7. Agreed Next Steps
- Create official GitHub directories for documentation and meeting notes.
- Establish a consistent workflow for updates, reviews, and documentation.
- Improve cross-team understanding of who works on what through Discord and GitHub synchronization.

**Summary of Goals for Next Sprint:**
1. Increase documentation consistency (GitHub artifacts, meeting minutes, user stories).
2. Strengthen communication (weekly updates, Discord structure).
3. Plan before coding, but remain flexible and agile.
4. Record experimental progress for shared learning.
5. Maintain focus and structured collaboration.

---

**Reflection:**
The group identified strong engagement and motivation but acknowledged weaknesses in structured planning, documentation, and communication flow. The next sprint will focus on developing consistent habits for transparency, coordination, and clarity.
